# Тест-кейсы для микросервиса объявлений

## Позитивные тест-кейсы

### TC-001: Успешное создание объявления
**Предусловия:** Нет
**Шаги:**
1. Отправить POST запрос на /api/items с валидными данными
2. Проверить ответ
**Ожидаемый результат:**
- Status code: 200
- В ответе присутствуют все отправленные поля
- Поле "id" заполнено и не пустое
- Поле "createdAt" заполнено

### TC-002: Получение объявления по существующему ID
**Предусловия:** Создано объявление с известным ID
**Шаги:**
1. Отправить GET запрос на /api/items/{existing_id}
2. Проверить ответ
**Ожидаемый результат:**
- Status code: 200
- Данные в ответе соответствуют созданному объявлению

### TC-003: Получение всех объявлений по sellerId
**Предусловия:** Создано несколько объявлений с одинаковым sellerId
**Шаги:**
1. Отправить GET запрос на /api/items?sellerId={existing_seller_id}
2. Проверить ответ
**Ожидаемый результат:**
- Status code: 200
- В ответе массив items содержит все объявления данного продавца
- Количество объявлений соответствует созданным

### TC-004: Получение статистики по существующему itemId
**Предусловия:** Создано объявление с известным ID
**Шаги:**
1. Отправить GET запрос на /api/items/{existing_id}/stats
2. Проверить ответ
**Ожидаемый результат:**
- Status code: 200
- В ответе присутствуют поля: itemId, views, clicks
- views и clicks имеют числовые значения

## Негативные тест-кейсы

### TC-005: Создание объявления с невалидными данными
**Предусловия:** Нет
**Шаги:**
1. Отправить POST запрос на /api/items с невалидными данными (отсутствуют обязательные поля)
2. Проверить ответ
**Ожидаемый результат:**
- Status code: 400 или 422
